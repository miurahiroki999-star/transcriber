# AI文字起こし - セットアップ手順

会議・フォーラム・打ち合わせの音声をタイムコード付きテキストに変換するWebアプリです。
**長尺音声の自動分割処理に対応しています（v2）。**

---

## ファイル構成

**以下の構成のままGitHubにアップロードしてください。**

```
ai-transcribe/
├── index.html                    ← フロントエンド（UI）
├── netlify/
│   └── functions/
│       └── transcribe.js         ← バックエンド（OpenAI API呼び出し）
├── netlify.toml                  ← Netlify設定
└── README.md                     ← この手順書
```

> ⚠️ `transcribe.js` は必ず `netlify/functions/` フォルダの中に置いてください。
> ルート直下に置いた場合、Netlify Functionsとして認識されず動作しません。

## 自動分割モードについて（重要）

### 概要

長尺音声（会議録音・フォーラム録音など）を、ブラウザ上で自動分割して順番に文字起こしする機能です。
**1本の音声ファイルを選ぶだけで、アプリ側が分割・送信・結合を自動的に行います。**

### 使い方

1. 音声ファイルを選択する
2. 「自動分割モード」がONになっていることを確認する（初期値はON）
3. 分割単位を選ぶ（初期値：90秒）
4. 「文字起こしを開始する」ボタンを押す
5. 分割・処理の進捗がリアルタイムで表示される
6. 完了後、タイムコード付きの結合テキストが表示される

### 初期値が90秒の理由

Netlify Functionsのタイムアウト制限（有料プランで最大26秒）に当たりにくくするため、短めに設定しています。
90秒のモノラル16kHz WAVチャンクは約2.8MBになります。環境によってはタイムアウトする場合があります。

**失敗する場合は60秒に変更してください。**

### 分割の仕組み（技術情報）

- ブラウザ内蔵の **Web Audio API** を使って音声をデコードします（追加ライブラリ不要）
- 指定秒数ごとに AudioBuffer を切り出します
- **モノラル 16kHz 16bit WAV** にエンコードして送信します
  - モノラル化・ダウンサンプリングにより、文字起こし用途で十分な音質を保ちながらファイルサイズを削減します
- チャンクは **1つずつ順番に送信**します（並列送信なし）
- 各チャンクの文字起こし結果に **チャンク開始秒を加算**してタイムコードを補正します

### タイムコード補正の仕組み

チャンク内タイムコードは 00;00;00;00 から始まるため、以下のように補正します：

```
チャンク1（0〜90秒）: 内部 00;00;00;00 → 出力 00;00;00;00
チャンク2（90〜180秒）: 内部 00;00;00;00 → 出力 00;01;30;00
チャンク3（180〜270秒）: 内部 00;00;00;00 → 出力 00;03;00;00
```

### 制限事項・注意点

- **処理時間について**：長尺音声（60分など）は全処理に数分かかります。ブラウザを閉じないでください
- **PCの性能について**：ブラウザ上で音声をデコード・エンコードするため、古いPCでは時間がかかる場合があります
- **対応形式について**：mp3・wav・mp4は通常問題ありませんが、m4aはブラウザによってはdecodeAudioDataで失敗する場合があります。その場合はmp3に変換してから試してください
- **Netlify Functionsの制限**：チャンクが大きすぎると失敗する場合があります。まず90秒で試し、失敗する場合は60秒に変更してください
- **API費用について**：コストはチャンク数ではなく音声の総時間に応じて発生します。60分の音声は60分分の料金がかかります。下記「概算コスト目安」を参照してください

## タイムスタンプについて（重要）

| モデル | タイムコード | 説明 |
|---|---|---|
| **whisper-1** | ✅ セグメント単位（最も正確） | APIから実際のタイムスタンプを取得。自動分割時もチャンク内タイムスタンプを補正して使用 |
| gpt-4o-transcribe | ⚠️ 推定値（目安） | APIの仕様上タイムスタンプ非対応。チャンク内を均等分割して算出し、チャンク開始秒を加算 |
| gpt-4o-mini-transcribe | ⚠️ 推定値（目安） | 同上 |

**タイムコード精度を重視する場合は whisper-1 を推奨します。**

---

## ファイルサイズについて

- **通常モード（自動分割OFF）**：推奨5MB以下、上限25MB
- **自動分割モード（自動分割ON）**：手動分割なしで長尺音声を処理できます。ただし、ブラウザ上で音声全体を読み込んで分割するため、PC性能・ブラウザ・音声形式・ファイルサイズによっては失敗する場合があります。最初は10〜30分程度の音声でテストしてください。失敗する場合はmp3に変換してから再試行してください。
- 自動分割モードでは、ブラウザ上で音声を分割してから送るため、1回の送信は最大約2〜5MBに収まります

## Step 1：GitHubにアップロード

1. [github.com](https://github.com) でアカウントを作成（未作成の場合）
2. 右上の「**＋**」→「**New repository**」をクリック
3. Repository name に `ai-transcribe`（任意）と入力
4. 「**Private**」を選択（APIキー関連のファイルを含むため）
5. 「**Create repository**」をクリック
6. 「**uploading an existing file**」をクリック
7. 以下のファイル・フォルダをドラッグ＆ドロップ（フォルダ構造を保ったまま）：
   - `index.html`
   - `netlify/functions/transcribe.js`（フォルダごとドロップ）
   - `netlify.toml`
8. 「**Commit changes**」をクリック

> ⚠️ `netlify/functions/transcribe.js` は必ず `netlify/functions/` フォルダの中に置くこと。

---

## Step 2：Netlifyにデプロイ

1. [netlify.com](https://netlify.com) でアカウントを作成（未作成の場合）
2. ダッシュボードの「**Add new site**」→「**Import an existing project**」をクリック
3. 「**Deploy with GitHub**」を選択
4. GitHubアカウントを連携し、`ai-transcribe` リポジトリを選択
5. 設定画面では以下を確認：
   - **Branch to deploy**: `main`
   - **Build command**: （空欄のまま）
   - **Publish directory**: `.`（ドット1文字）
6. 「**Deploy site**」をクリック
7. デプロイ完了後、`https://ランダム名.netlify.app` のURLが表示される

---

## Step 3：環境変数（APIキー）の設定

> ⚠️ これを設定しないと文字起こしが動きません。

1. Netlifyダッシュボードで、デプロイしたサイトを開く
2. 上部メニューの「**Site settings**」をクリック
3. 左サイドバーの「**Environment variables**」をクリック
4. 「**Add a variable**」をクリック
5. 以下のように入力：
   - **Key**: `OPENAI_API_KEY`
   - **Value**: `sk-proj-xxxxxxxxxxxxx`（OpenAIのAPIキー）
6. 「**Save**」をクリック
7. 左サイドバーの「**Deploys**」をクリック
8. 「**Trigger deploy**」→「**Deploy site**」をクリックして再デプロイ

### OpenAI APIキーの取得方法

1. [platform.openai.com](https://platform.openai.com) にログイン
2. 左メニュー「**API Keys**」→「**Create new secret key**」
3. 作成されたキー（`sk-proj-...`）をコピー
4. ⚠️ キーは一度しか表示されないのでメモしておくこと

## Step 4：動作確認テスト手順

**必ずこの順番でテストしてください。**

### ステップ1：短い音声（通常モード）でテスト
自動分割をOFFにして、30秒〜1分のm4aをアップロードし、基本動作を確認します。

### ステップ2：短い音声（自動分割モード）でテスト
自動分割をONにして、同じファイルで試します。進捗表示（チャンクバッジ）が出ることを確認します。

### ステップ3：長尺音声（10〜30分）で自動分割テスト
分割単位90秒で試します。処理中に「全N チャンク中 M/N を文字起こし中」と表示されることを確認します。

### ステップ4：whisper-1でタイムコード精度を確認
whisper-1を選択して実行し、タイムコードが実際の発話タイミングに近いか確認します。

### ステップ5：失敗したチャンクの確認
（意図的に失敗させる必要はありません）失敗チャンクがある場合、結果欄の下部に「失敗したチャンク」が一覧表示されることを確認します。

## 想定エラーと対処

| エラー / 症状 | 原因 | 対処方法 |
|---|---|---|
| `Function not found` / 404 | transcribe.js のパスが違う | `netlify/functions/transcribe.js` の場所を確認 |
| 「APIキーが設定されていません」 | 環境変数未設定 | Netlify > Site settings > Environment variables で設定 |
| 「APIキーが無効」 | キーが間違っている / 期限切れ | OpenAIダッシュボードでキーを再発行 |
| 「ブラウザで音声を分割できませんでした」 | m4a等の形式がブラウザ非対応 | mp3に変換してから試す |
| チャンクが失敗する | Netlify Functionsのタイムアウト / チャンクが大きい | 分割単位を90秒→60秒に変更する |
| タイムアウトで失敗する | Netlify無料プランの制限（10秒） | 有料プランへアップグレード（最大26秒）|
| CORS エラー | 設定ミス | netlify.toml の CORS ヘッダー設定を確認 |
| タイムコードがズレている | gpt-4o系モデルの推定タイムコード | whisper-1 を使用する |
| 日本語が文字化け | txtファイルのエンコーディング | メモ帳でUTF-8として開く |
| デプロイが失敗する | netlify.tomlの設定ミス | `publish = "."` になっているか確認 |
| 処理が長くて途中で止まった | ブラウザがスリープ / タブが非アクティブ | 処理中はブラウザを閉じない。タブをアクティブにしておく |
