# AI文字起こし - セットアップ手順

会議・フォーラム・打ち合わせの音声をタイムコード付きテキストに変換するWebアプリです。

---

## ファイル構成

**以下の構成のままGitHubにアップロードしてください。**

```
ai-transcribe/
├── index.html                    ← フロントエンド（UI）
├── netlify/
│   └── functions/
│       └── transcribe.js         ← バックエンド（OpenAI API呼び出し）
├── netlify.toml                  ← Netlify設定
└── README.md                     ← この手順書
```

> ⚠️ `transcribe.js` は必ず `netlify/functions/` フォルダの中に置いてください。
> ルート直下に置いた場合、Netlify Functionsとして認識されず動作しません。

---

## タイムスタンプについて（重要）

| モデル | タイムコード | 説明 |
|---|---|---|
| **whisper-1** | ✅ セグメント単位（最も正確） | APIから実際のタイムスタンプを取得 |
| gpt-4o-transcribe | ⚠️ 推定値（目安） | APIの仕様上タイムスタンプ非対応。音声時間を均等分割して算出 |
| gpt-4o-mini-transcribe | ⚠️ 推定値（目安） | 同上 |

正確なタイムコードが必要な場合は **whisper-1** を選択してください。
gpt-4o系モデルで文字起こしした場合、結果欄に「このタイムコードは推定です」と表示されます。

---

## ファイルサイズについて（重要）

- **推奨：5MB以下**
- OpenAI APIの上限は25MBですが、本アプリはブラウザでBase64変換してNetlify Functionへ送信するため、Netlify側のリクエストサイズ上限やタイムアウトに先に当たる場合があります。
- **まず1〜3分程度の短い音声でテストしてください。**
- **長尺音声は失敗する可能性があります。**

## Step 1：GitHubにアップロード

1. [github.com](https://github.com) でアカウントを作成（未作成の場合）
2. 右上の「**＋**」→「**New repository**」をクリック
3. Repository name に `ai-transcribe`（任意）と入力
4. 「**Private**」を選択（APIキー関連のファイルを含むため）
5. 「**Create repository**」をクリック
6. 「**uploading an existing file**」をクリック
7. 以下のファイル・フォルダをドラッグ＆ドロップ（フォルダ構造を保ったまま）：
   - `index.html`
   - `netlify/functions/transcribe.js`（フォルダごとドロップ）
   - `netlify.toml`
8. 「**Commit changes**」をクリック

> ⚠️ `netlify/functions/transcribe.js` は必ず `netlify/functions/` フォルダの中に置くこと。

---

## Step 2：Netlifyにデプロイ

1. [netlify.com](https://netlify.com) でアカウントを作成（未作成の場合）
2. ダッシュボードの「**Add new site**」→「**Import an existing project**」をクリック
3. 「**Deploy with GitHub**」を選択
4. GitHubアカウントを連携し、`ai-transcribe` リポジトリを選択
5. 設定画面では以下を確認：
   - **Branch to deploy**: `main`
   - **Build command**: （空欄のまま）
   - **Publish directory**: `.`（ドット1文字）
6. 「**Deploy site**」をクリック
7. デプロイ完了後、`https://ランダム名.netlify.app` のURLが表示される

---

## Step 3：環境変数（APIキー）の設定

> ⚠️ これを設定しないと文字起こしが動きません。

1. Netlifyダッシュボードで、デプロイしたサイトを開く
2. 上部メニューの「**Site settings**」をクリック
3. 左サイドバーの「**Environment variables**」をクリック
4. 「**Add a variable**」をクリック
5. 以下のように入力：
   - **Key**: `OPENAI_API_KEY`
   - **Value**: `sk-proj-xxxxxxxxxxxxx`（OpenAIのAPIキー）
6. 「**Save**」をクリック
7. 左サイドバーの「**Deploys**」をクリック
8. 「**Trigger deploy**」→「**Deploy site**」をクリックして再デプロイ

### OpenAI APIキーの取得方法

1. [platform.openai.com](https://platform.openai.com) にログイン
2. 左メニュー「**API Keys**」→「**Create new secret key**」
3. 作成されたキー（`sk-proj-...`）をコピー
4. ⚠️ キーは一度しか表示されないのでメモしておくこと

## Step 4：動作確認テスト手順

**必ずこの順番でテストしてください。**

### ステップ1：30秒〜1分のm4aでテスト
短い音声（推奨：1MB以下）をアップロードし、文字起こしが完了することを確認します。
まずここで基本動作を確かめてください。

### ステップ2：1〜3分のmp3でテスト
少し長い音声でも動作するか確認します。

### ステップ3：whisper-1でタイムコード精度を確認
モデルを「whisper-1」に切り替えて実行し、タイムコードが実際の発話タイミングに沿っているか確認します。疑似タイムコードの注記が**表示されないこと**を確認してください。

### ステップ4：gpt-4o-mini-transcribeで文字起こし精度を確認
モデルを「gpt-4o-mini-transcribe」に切り替えて実行し、文字起こし精度を確認します。結果欄に「このタイムコードは推定です」という注記が**表示されること**を確認してください。

### ステップ5：5MB超の音声で警告表示を確認
5MBを超えるファイルを選択した際に、黄色い警告メッセージが表示されることを確認します（実行は拒否されない）。

### ステップ6：25MB超の音声で拒否表示を確認
25MBを超えるファイルを選択した際に、赤いエラーメッセージが表示されて実行できないことを確認します。

## 想定エラーと対処

| エラー / 症状 | 原因 | 対処方法 |
|---|---|---|
| `Function not found` / 404 | transcribe.js のパスが違う | `netlify/functions/transcribe.js` の場所を確認 |
| 「APIキーが設定されていません」 | 環境変数未設定 | Netlify > Site settings > Environment variables で設定 |
| 「APIキーが無効」 | キーが間違っている / 期限切れ | OpenAIダッシュボードでキーを再発行 |
| 「ファイルサイズが大きすぎます」 | 25MB超 | 音声を分割する |
| タイムアウトで失敗する | Netlify無料プランの制限（10秒）または長尺音声 | 短い音声で試す。有料プランにアップグレードすると最大26秒まで延長可能 |
| CORS エラー | 設定ミス | netlify.toml の CORS ヘッダー設定を確認 |
| タイムコードがズレている | gpt-4o系モデルの推定タイムコード | whisper-1 を使用する |
| 日本語が文字化け | txtファイルのエンコーディング | メモ帳でUTF-8として開く |
| デプロイが失敗する | netlify.tomlの設定ミス | `publish = "."` になっているか確認 |

## 検品チェックリスト

デプロイ後、以下をすべて確認してください。

### 基本表示
- [ ] サイトが `https://xxxx.netlify.app` で開く
- [ ] 「AI文字起こし」タイトルが表示されている
- [ ] ファイル選択エリア（ドラッグ＆ドロップ対応）がある
- [ ] モデル選択プルダウンがある（3種類）
- [ ] 各モデルの説明に「タイムコードは目安」「タイムコード精度優先」の表記がある

### ファイル選択
- [ ] m4aを選択すると「✅ ファイル名.m4a（○.○MB）」と表示される
- [ ] 推定音声時間と概算コストが表示される
- [ ] モデルを変えるとコストが変わる
- [ ] **5MB超のファイルを選択すると黄色い警告が出る（拒否はされない）**
- [ ] **25MB超のファイルを選択すると赤いエラーが出て選択が無効になる**
- [ ] 対応外形式（.docxなど）を選択すると日本語エラーが出る

### 文字起こし実行
- [ ] ファイルを選ばずに「実行」すると「ファイルが選択されていません」が出る
- [ ] 実行中に「文字起こし中です...」と経過秒数が表示される
- [ ] 結果が表示される
- [ ] タイムコードが `00;00;00;00` 形式で付いている

### gpt-4o系のタイムコード注記
- [ ] gpt-4o-mini-transcribe で実行すると「このタイムコードは推定です」の注記が表示される
- [ ] whisper-1 で実行すると上記注記が**表示されない**

### 結果・後処理
- [ ] 「全文コピー」で「✅ コピーしました」が出る
- [ ] 「txtで保存」でダウンロードできる
- [ ] ダウンロードしたファイルをメモ帳で開くと文字化けしていない
